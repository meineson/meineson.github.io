    • AI Agent使用MCP调用Pixso、Figma进行前端UI界面自动设计

    为了让AI Agent编程工具不局限与命令行代码生成和调试，可以通过MCP扩展其能力，例如通过MCP调用Chrome浏览器直接调试AI生成的WEB代码，不用手工去浏览器里截图和复制报错来告诉AI修bug或改代码了，它可以自动验证。

    那么理论上，也可以直接通过MCP调用Figma、Pixso这些UI设计工具来辅助软件界面设计，这样就不需要美工了（笑），实战一下？

    从互联网新闻的碎片中可以了解到，最先拥抱AI的是Figma，然后Pixso这些国产平替们也加入了AI功能，但都是类似WPS+AI这种在自己的一亩三分地里倒腾，给人的感觉就是……能用，但不好用。

    所以最新一个新闻里，有个国外小哥用Claude Code+MCP调用Figma，而不是用Figma自带的AI设计功能，完全颠覆了使用体验，我们来求证一下。

    由于Figma免费版本不开放MCP需要付费订阅，而Pixso免费版本就可以启用MCP，先用Pixso测试，开启方式很简单，在菜单里有明显的Pixso MCP开关，打开就行，然后在Opencode或Claude Code里添加MCP服务。

    OpenCode添加Pixso MCP

    Claude Code添加Pixso MCP

    重启OC或CC就可以指示AI设计一个具体的软件界面了，例如：用Pixso设计一个WebRTC视频会议UI。

    结果试了好多次各种卡bug，最后大模型的这段Thinking看完我要骂人了。

    什么意思？意思是，Pixso通过MCP只开放了大量get（也就是只读权限），写权限有哪些？创建组件、给组件应用样式……嗯，拜拜了您，Pixso还是护食的，你想通过AI大模型指导它直接修改UI设计稿没戏，这是它的吃饭家伙不开放。
    它只希望你用AI Agent工具读取它的设计稿来生成HTML代码，不希望AI Agent插手它的设计工作。官网的帮助文档示例也验证了这一点，它希望你的用法是：“选中Pixso里的一个设计容器，让AI Agent生成对应的HTML代码。”

    再确认下Figma官方的MCP的能力，果然也是类似的，是“只读”的，官方希望你用AI将它的UI设计稿转成代码，而不希望你用AI来直接设计UI。

    但是，话怎么说的？你不干，有的是人干，Figma有开放API，所有就有社区版本的Figma MCP服务来解决这个问题。
    例如这个：https://github.com/arinspunk/claude-talk-to-figma-mcp

    并且这个社区版本MCP不需要付费Figma，用免费的Figma就可以工作，我们来实战吧。
